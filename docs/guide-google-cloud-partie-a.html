Guide Google Cloud — Partie A

Configuration admin (une fois) pour la synchronisation Google Calendar ↔ Centre Est

Projet : centre-est-calendar · Scope : calendar

0 Avant de commencer

Préparez les éléments suivants :

  • Un compte Google avec accès à la Google Cloud Console
  • La liste des emails Google des médecins testeurs (ex. dr.xxx@gmail.com)
  • Les URLs de l’application :
    • Local : backend http://localhost:4000, frontend http://localhost:5173
    • Production : ex. https://chennoufi.nav.ovh

1 Créer ou ouvrir le projet

  1. Ouvrir console.cloud.google.com
  2. Cliquer sur le sélecteur de projet (en haut à gauche)
  3. Nouveau projet → nom : centre-est-calendar → Créer
    ou sélectionner le projet existant
  4. Vérifier que le nom du projet actif est correct dans la barre du haut

2 Activer l’API Google Calendar

  1. Menu ☰ → APIs & Services → Library (Bibliothèque)
  2. Rechercher : Google Calendar API
  3. Ouvrir Google Calendar API
  4. Cliquer sur Enable (Activer)
Important : sans cette étape, la liaison agenda échouera (API non activée).

3 Écran de consentement OAuth

Écran affiché au médecin lors du clic « Lier Google Calendar ».

3.1 Démarrer

  1. Menu ☰ → APIs & Services → OAuth consent screen
  2. User Type :
    • External — médecins avec Gmail personnels ou comptes hors Workspace
    • Internal — uniquement si tous les médecins sont sur le même Google Workspace
  3. Choisir External (cas le plus courant) → Create

3.2 Informations sur l’application

ChampExemple

User support emailEmail admin

Authorized domains (prod)chennoufi.nav.ovh (sans https://)

→ Save and Continue

3.3 Scopes (autorisations)

  1. Add or Remove Scopes
  2. Ajouter uniquement ce scope :
https://www.googleapis.com/auth/calendar

Libellé typique : See, edit, share, and permanently delete all calendars…

→ Update → Save and Continue

Ne pas ajouter Gmail, Drive, etc. — cela complique la validation Google.

3.4 Test users (obligatoire en mode Testing)

  1. Add users
  2. Ajouter chaque email Google des médecins qui testeront :
    • Email du Dr testeur
    • Email admin
    • Tout compte qui cliquera « Lier Google Calendar »
  3. Save and Continue
Si un email n’est pas dans la liste : erreur Access blocked / not a test user.

3.5 Résumé

→ Back to Dashboard

Vérifier : Publishing status = Testing · Scope calendar · Test users renseignés.

4 Identifiants OAuth (Client ID + Secret)

4.1 Créer le client Web

  1. APIs & Services → Credentials
  2. + Create Credentials → OAuth client ID
  3. Application type : Web application
  4. Name : Centre Est — Web (local + prod)

4.2 Authorized redirect URIs

Ajouter exactement (copier-coller, sans espace ni slash en trop) :

Développement / PC test

http://localhost:4000/api/medecin/google/callback

Production (adapter le domaine)

https://chennoufi.nav.ovh/api/medecin/google/callback

4.3 Authorized JavaScript origins (recommandé)

http://localhost:5173
https://chennoufi.nav.ovh

→ Create

4.4 Copier les identifiants

Élément GoogleVariable backend/.env
Client IDGOOGLE_CLIENT_ID
Client secretGOOGLE_CLIENT_SECRET
Le secret n’est affiché qu’une fois. Ne jamais le committer dans Git. Utiliser Reset secret si perdu.

5 Variables d’environnement backend

Fichier backend/.env — aligner avec Google Console.

Local

GOOGLE_CLIENT_ID=votre-client-id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=GOCSPX-votre-secret
GOOGLE_REDIRECT_URI=http://localhost:4000/api/medecin/google/callback
FRONTEND_URL=http://localhost:5173

Production

GOOGLE_REDIRECT_URI=https://chennoufi.nav.ovh/api/medecin/google/callback
FRONTEND_URL=https://chennoufi.nav.ovh
Règle : GOOGLE_REDIRECT_URI doit être identique à une ligne « Authorized redirect URIs » dans Google Cloud.

Puis redémarrer le backend : npm run dev

6 Testing vs Production (Google)

ModeQui peut se connecterAction
Testing Uniquement les Test users (étape 3.4) Ajouter chaque email médecin
In production Tout compte Google (après vérification) Publier l’app + validation Google

Pour les tests sur le PC du médecin : rester en Testing et lister son Gmail.

7 Checklist admin

  • Projet Google Cloud créé ou sélectionné
  • Google Calendar API activée
  • OAuth consent screen configuré (External + scope calendar)
  • Tous les emails médecins testeurs ajoutés en Test users
  • OAuth client Web créé (type Web application)
  • Redirect URI local + prod ajoutées
  • Client ID et Secret copiés dans backend/.env
  • Backend redémarré après modification du .env

Dépannage rapide

ErreurSolution
redirect_uri_mismatch Aligner GOOGLE_REDIRECT_URI et Google Console (caractère par caractère)
Access blocked / not a test user Ajouter l’email du médecin dans OAuth consent screen → Test users
invalid_client Vérifier Client ID et Secret dans .env
API not enabled Library → activer Google Calendar API

Suite : Partie B (dans l’application)

Après la Partie A, sur le PC du médecin :

  1. Ouvrir Centre Est → Agenda médecin
  2. Sélectionner le médecin concerné
  3. Cliquer Lier Google Calendar
  4. Se connecter avec le même compte Google ajouté en Test user
  5. Accepter l’accès au calendrier → badge Connecté

La synchronisation est ensuite automatique (plus de bouton Synchroniser manuel).
